# Contributing to cross-review-mcp

Thanks for your interest in contributing!

## Code of Conduct

Be respectful. We build tools for critical thinking.

## Report a Bug

Search existing issues first. If not found, open a new issue with your Node.js version and MCP client details.

## Suggest a Feature

Open an issue labeled 'enhancement' describing the use case.

## Submit a PR

1. Create a branch: git checkout -b fix/your-issue
2. Make changes in src/
3. Add tests in test/
4. Run: npm run build && npm test
5. Commit with clear message

## Code Style

Use TypeScript, follow naming conventions, use Zod for validation, add JSDoc comments.

## Testing

Add tests for new features. Write tests for happy paths AND error cases.

Thanks for contributing!

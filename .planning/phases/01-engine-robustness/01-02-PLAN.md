**Update parseVerdict() regex patterns:**

Replace the current regex patterns with markdown-aware versions. The key insight: strip `**` from the string before matching, or make the regex optionally match `**`.

Preferred approach — strip markdown bold before matching (simpler, more maintainable):

```typescript
export function parseVerdict(summary: string): "proceed" | "revise" | "abort" {
  // Strip markdown bold markers for consistent matching
  const cleaned = summary.replace(/\*\*/g, "");

  // Try explicit VERDICT: line first (most reliable)
  const verdictMatch = cleaned.match(
    /^\s*(?:OVERALL\s+)?VERDICT:\s*(PROCEED|REVISE|ABORT)/im
  );
  if (verdictMatch) {
    return verdictMatch[1].toLowerCase() as "proceed" | "revise" | "abort";
  }

  // Fallback: search in summary/conclusion/recommendation section
  const lowerCleaned = cleaned.toLowerCase();
  const summarySection =
    lowerCleaned.split(/summary:|conclusion:|recommendation:/i).pop() || "";

  if (summarySection.includes("abort")) {
    return "abort";
  } else if (summarySection.includes("proceed")) {
    return "proceed";
  }

  return "revise";
}
```

Key changes:
1. Strip `**` globally before any matching — handles all bold variants in one step
2. Combine the two separate regex patterns (VERDICT: and OVERALL VERDICT:) into one with optional `OVERALL\s+` group
3. Allow leading whitespace with `^\s*`
4. Keep the /im flags (case-insensitive, multiline)

**Update countHighConfidenceClaims() regex:**

```typescript
export function countHighConfidenceClaims(critique: string): number {
  // Strip markdown bold markers for consistent matching
  const cleaned = critique.replace(/\*\*/g, "");
  const matches = cleaned.match(/Confidence:\s*HIGH/gi);
  return matches ? matches.length : 0;
}
```

Key changes:
1. Strip `**` before matching — handles `**Confidence: HIGH**`, `**Confidence:** HIGH`, `Confidence: **HIGH**`
2. Keep the existing `/gi` flags (global, case-insensitive) — this already handles case variations
3. Use `\s*` instead of `\s` to handle zero or multiple spaces between colon and HIGH

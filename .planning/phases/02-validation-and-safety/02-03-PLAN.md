<task type="auto">
  <name>Task 2: GREEN — Implement token estimation and size guards in review()</name>

1. In `src/engine.ts`, replace the `estimateTokens` stub with a real implementation:

   ```typescript
   /**
    * Rough token estimate: ~4 characters per token (GPT tokenizer approximation).
    * Not precise, but sufficient for pre-flight size gating.
    */
   export function estimateTokens(text: string): number {
     return Math.ceil(text.length / 4);
   }
   ```

   This is intentionally simple. The requirements say "approximate" (~50K, ~100K). A character-based heuristic is standard practice and matches the project's "nanoseconds vs 30s API calls" philosophy (no need for a real tokenizer).

2. Define threshold constants near the top of engine.ts (after MODEL_COSTS):

   ```typescript
   const TOKEN_WARN_THRESHOLD = 50_000;    // SAFE-01: warn above this
   const TOKEN_REJECT_THRESHOLD = 100_000; // SAFE-02: reject above this
   ```

3. In the `review()` method, add size checking at the very top (after `const contentType = ...` line, before `const levelConfig = ...`):

   ```typescript
   // Content size pre-flight check
   const estimatedTokens = estimateTokens(content);
   if (estimatedTokens > TOKEN_REJECT_THRESHOLD) {
     throw new Error(
       `Content too large: ~${estimatedTokens.toLocaleString()} tokens (estimated). ` +
       `Maximum is ~${TOKEN_REJECT_THRESHOLD.toLocaleString()} tokens. ` +
       `Reduce content size before submitting for review.`
     );
   }
   ```

   This goes BEFORE any API call setup — the throw prevents wasted work.

4. For the warning case, add after the rejection check but still before API calls:

   ```typescript
   let warning: string | undefined;
   if (estimatedTokens > TOKEN_WARN_THRESHOLD) {
     warning = `Large content: ~${estimatedTokens.toLocaleString()} estimated tokens. ` +
       `Review will proceed, but costs may be elevated and some models may timeout.`;
   }
   ```

5. In the return statement of `review()`, add the `warning` field:

   ```typescript
   return {
     contentLength: content.length,
     // ... existing fields ...
     warning,
   };
   ```

   Note: Only include `warning` if it's defined (TypeScript will omit undefined optional fields in the object literal, keeping the response clean for normal-sized content).

6. Build and run tests: `npm test`. All tests should pass.

1

What Is This?

When you have a question — a bug in your code, a design decision, a technical problem — you normally ask one AI model (like ChatGPT or Gemini). You get one answer and hope it's right.

Cross-Review MCP lets you ask 5 different AI models the same question at the same time. It takes the same 3–5 seconds as asking one, because they all work in parallel.

Then it shows you:

  • ✓ What they all agree on (the consensus)
  • ✧ What each model individually thinks
  • ⚠ Where they disagree (which is actually useful!)
  • % A confidence score — how sure they are

You have 5 expert friends. You text all of them the same question at once. In 3 seconds, they all reply. 4 out of 5 say the same thing — now you're pretty confident in the answer. The 5th friend has a different idea — worth thinking about too!

Same work, but all happening at the same time — so it's as fast as the slowest single model

Installation

Three steps. That's it.

1

Install the package

Open your terminal and run this command. It downloads the software from npm (a big library of tools).

Terminal
npm install -g cross-review-mcp
2

Get API keys

Each AI company gives you a "key" (like a password) so you can use their service. Sign up at any of these:

  • • OpenAI — for GPT-5.2
  • • Google AI Studio — for Gemini
  • • DeepSeek
  • • Mistral
  • • OpenRouter — gives access to many models

Don't have all 5? That's totally fine. Even 2 keys will work. The more you have, the better the consensus.

3

Create a .env file

In your project folder, create a file called .env and paste your keys:

.env
# Paste your keys here (use whichever ones you have) OPENAI_API_KEY=sk-proj-your-key-here GEMINI_API_KEY=AIzaSy-your-key-here DEEPSEEK_API_KEY=sk-your-key-here MISTRAL_API_KEY=your-key-here OPENROUTER_API_KEY=sk-or-v1-your-key-here
4

How to Use It

Three ways to use cross-review, depending on how you like to work:

Option A: Command Line

Just type a command in your terminal. Great when you're already coding:

Terminal
# Ask about a bug you're stuck on cross-review dev \ --error "PORT IS IN USE at 6277" \ --tech "Node.js" \ --env "macOS" \ --tried "Killed the process, restarted"

Option B: Claude Desktop

If you use Claude Desktop (the desktop app from Anthropic), cross-review works inside it automatically. Add this to your MCP config:

claude_desktop_config.json
{ "mcpServers": { "cross-review": { "command": "npx", "args": ["cross-review-mcp"] } } }

Then just ask Claude naturally: "Can you cross-review this code for security issues?"

Option C: Web Dashboard

Start the server with a visual dashboard so you can watch the models work in real time:

Terminal
# Start the dashboard npm run serve # Then open your browser to: # http://localhost:6280

6

What You Can Do (4 Tools)

Cross-review exposes 4 tools you can call:

get_dev_guidance
Stuck on a bug? Describe your error and get 5 expert opinions plus a consensus fix.
  • error required — the error message or problem
  • technology required — what tech you're using (Node.js, Docker, etc.)
  • environment required — your OS (macOS, Linux, Windows)
  • attempts required — what you've already tried
review_content
Submit code or text for peer review. Each model checks for different issues.
  • content required — your code or text to review
  • reviewType required — security, performance, correctness, style, or general
get_cache_stats
See how many questions were answered from memory (cached) vs. needing new API calls.
get_cost_summary
Check how much you've spent on API calls today, this month, and per model.
7

Three Ways to Run the Server

Mode Command What It Does Best For
stdio npm start Talks directly to Claude Desktop through a pipe Local use with Claude
http npm run serve Web server with dashboard you can see in a browser Dashboard + remote access
both npm run serve:both Both at the same time — local Claude + web dashboard Maximum flexibility

stdio is like a walkie-talkie — direct, private communication between your tool and the server. http is like a website — anyone on your network can open it in a browser. both is having both the walkie-talkie and the website at once.

Common Questions

"Error: Missing API key"
Create a .env file with your API keys. See Step 3 in the Installation section above.
"No model responses"
Check that your API keys are correct, haven't expired, and that you have internet access. Try running node test/test-keys.mjs to verify.
"The answer isn't helpful"
Give more context! The more details you provide (technology, OS, what you already tried), the better the answers. Like asking a doctor — "it hurts" is less useful than "my left knee hurts when I run."
"Do I need all 5 API keys?"
No! Even 2 keys will work. The system uses whatever keys you provide. More models = better consensus, but any combination works.
"Is this free?"
The software itself is free and open source. But the AI services charge per question (see Cost & Speed above). Gemini is practically free, OpenAI is the most expensive at ~$0.03 per question.

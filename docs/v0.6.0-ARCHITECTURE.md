## Feature Integration Flow

### Single Review Request (with caching + rate limiting)

```
User submits single review: "Review this code..."
         │
         ▼
┌─────────────────────────────────────────┐
│ EXECUTOR.execute(request)               │
└─────────────────────────────────────────┘
         │
         ▼
    ┌─────────────────────────────────────┐
    │ CACHE CHECK                         │
    ├─────────────────────────────────────┤
    │ hash = SHA256(code)                 │
    │ if (cache.has(hash)) {              │
    │   return cache.get(hash)  ← INSTANT │
    │ }                                   │
    └─────────────────────────────────────┘
         │
         │ CACHE MISS → Continue
         ▼
    ┌─────────────────────────────────────┐
    │ RATE LIMITER CHECK                  │
    ├─────────────────────────────────────┤
    │ For OpenAI:                         │
    │  if (tokens < 1) {                  │
    │    wait(timeUntilTokenReady)        │
    │  }                                  │
    │ tokens--                            │
    │                                     │
    │ For Gemini:                         │
    │  if (tokens < 1) {                  │
    │    wait(timeUntilTokenReady)        │
    │  }                                  │
    │ tokens--                            │
    └─────────────────────────────────────┘
         │
         ▼
    ┌─────────────────────────────────────┐
    │ CALL PROVIDERS (parallel)           │
    ├─────────────────────────────────────┤
    │ Promise.all([                       │
    │   openai.sendRequest(content),      │
    │   gemini.sendRequest(content)       │
    │ ])                                  │
    └─────────────────────────────────────┘
         │
         ▼
    ┌─────────────────────────────────────┐
    │ CACHE RESULT                        │
    ├─────────────────────────────────────┤
    │ cache.set(hash, result, {           │
    │   ttl: 86400  // 24 hours           │
    │ })                                  │
    └─────────────────────────────────────┘
         │
         ▼
Return result to user
```

## How Components Interact

### Scenario 1: Batch of 100 with 30 Duplicates

```
Batch: [review_1, review_2, ..., review_100]
       (70 unique, 30 duplicates of the 70)

Processing:
  │
  ├─ Review 1: CACHE MISS → API call → CACHE + RATE LIMIT
  │
  ├─ Review 2: CACHE MISS → API call → CACHE + RATE LIMIT
  │
  ├─ Review 3: CACHE HIT → Return cached result (instant, $0)
  │
  ├─ Review 4: CACHE HIT → Return cached result (instant, $0)
  │
  └─ ... (pattern repeats)

Results:
  Total API calls: 70 (not 100!)
  Cost: 70 × $0.0049 = $0.343 (saved ~$0.147)
  Time: ~30 seconds with 4 workers (vs 130 seconds sequential)
  Rate limit: Never exceeded (rate limiter prevents it)
```

### Scenario 2: Rate Limit Throttling

```
Minute 1: 100 reviews submitted
  │
  ├─ Worker 1 calls OpenAI
  │  OpenAI quota: 100/min
  │  Tokens available: 100 → 99 (after call)
  │
  ├─ Worker 2 calls OpenAI
  │  Tokens available: 99 → 98
  │
  ├─ ... (continues)
  │
  ├─ Worker 100 calls OpenAI
  │  Tokens available: 1 → 0
  │
  └─ Worker 101 calls OpenAI
     Tokens available: 0
     → Rate limiter: WAIT 0.6 seconds
     → Tokens refill to 1
     → Proceed with call ✅

Result: No 429 errors. Smooth processing. Takes slightly longer but reliable.
```

## Performance Impact Summary

### Single Request (1.3 seconds baseline)

```
Without v0.6.0:
  Request 1: 1,250ms + $0.0049
  Request 1 again: 1,250ms + $0.0049 (duplicate)
  Total: 2,500ms, $0.0098

With v0.6.0:
  Request 1: 1,250ms + $0.0049 (cache miss)
  Request 1 again: 5ms + $0 (cache hit)
  Total: 1,255ms, $0.0049
  → 50% cost savings, 50% time savings
```

### Batch of 100 (with 30 duplicates)

```
Without v0.6.0:
  Sequential: 100 × 1.3s = 130s
  Cost: 100 × $0.0049 = $0.49
  Risk: Rate limit errors at ~50% mark

With v0.6.0 (4 workers):
  Parallel: 25 batches × 1.3s = 32.5s
  Cost: 70 × $0.0049 = $0.343 (duplicates cached)
  Risk: Zero (rate limiter prevents quota exhaustion)
  
  Improvement:
  → 4x faster (130s → 32.5s)
  → 30% cheaper ($0.49 → $0.343)
  → 100% reliable (no errors)
```
